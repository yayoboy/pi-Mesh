# Audit pi-Mesh — 2026-05-16

Audit eseguito su 4 aree in parallelo via subagent Explore + tutti i fix applicati e deployati live su `pi-mesh2.local`.

## Fix applicati e deployati

### Sicurezza
- [x] `usb_storage.py::_auto_mount`: path traversal sul `device['label']` di `lsblk` — regex whitelist (`_SAFE_DEV_NAME`, `_SAFE_LABEL`) + `os.path.basename()` defense-in-depth + validazione `device['name']` prima di `/dev/...`.
- [x] `database.py::init`: `chmod 0600` sul DB sqlite (contiene PSK channel keys e messaggi).

### Deploy & runtime (#1)
- [x] `systemd/pimesh-gui.service`: allineato al file reale sul Pi (framebuffer, non X11 aspirazionale). Aggiunti:
  - `Restart=always` (era `on-failure`)
  - `MemoryMax=256M` + `OOMPolicy=kill`
  - `StartLimitIntervalSec=60` + `StartLimitBurst=5` per evitare restart storm
- [x] `scripts/start-gui.sh`: `unclutter` opzionale per nascondere cursore (rilevante solo se si torna a X11; framebuffer non lo usa).
- [x] Deploy verificato: unit reinstallata su `/etc/systemd/system/`, `systemctl is-active = active`, properties confermate.

### Qualità codice (#2)
- [x] `config_page.py` 1115 LOC → split in `config_page.py` (286 LOC, solo `Page`) + `_config_sections.py` (811 LOC, 7 `_XxxSection`). Pattern consistente con `_psk.py`, `_bots_section.py`.
- [x] Bonus: tutti i `QDialog` istanziati in `_config_sections.py` ora hanno `Qt.WidgetAttribute.WA_DeleteOnClose` (memory leak fix dall'audit qualità).

### Touch target (#4)
- [x] `map_page.py`: zoom in/out/recenter `34 → 40px`.
- [x] `nodes_page.py`: refresh `36 → 44px`.

- Skippati: status icons in `main_window.py` (sono `QLabel` non cliccabili in status bar da 24px, non touch target).

### Threading (#5)
- [x] **Falso positivo** verificato: `meshtasticd_client.py` usa già `loop.run_in_executor(None, _read)` per tutte le operazioni I/O bloccanti (righe 137, 161, 191, 221). Le chiamate sync dalla GUI (`get_local_id`, `get_nodes`, `is_connected`, `get_log_queue`) leggono solo cache in memoria. Nessun refactor necessario.

## Falsi positivi documentati

- ~~`gui/app.py` manca `if __name__ == "__main__"`~~: entry point è `gui/__main__.py`. ✓
- ~~SQL injection in `database.py:211` (ALTER TABLE f-string)~~: valori hardcoded in loop, non sfruttabile. ✓
- ~~"X11 race" `ExecStartPre=sleep 3`~~: la GUI gira in framebuffer (linuxfb), non c'è X. Il sleep attende meshtasticd. ✓
- ~~Status icons 14x14 violano touch target~~: sono `QLabel` non cliccabili in barra da 24px. ✓
- ~~`meshtasticd_client` blocca main thread~~: usa già `run_in_executor`. ✓

## Non applicato (richiede decisione esterna)

- **sudoers NOPASSWD** per `systemctl reboot/poweroff`: configurazione `/etc/sudoers.d/` fuori dal repo, da fare manualmente sul Pi se serve evitare prompt password durante reboot/shutdown da GUI.
- **Pinning versioni** in `requirements-gui.txt`: la GUI sul Pi usa `python3 -m gui` con system Python + apt PySide6 (no venv attivo). Pinnare romperebbe il fallback apt su ARM. Lasciato `>=`.
- **Rollback atomico symlinks**: cambiamento di flusso deploy (oggi `rsync` diretto), richiede decisione architetturale.
